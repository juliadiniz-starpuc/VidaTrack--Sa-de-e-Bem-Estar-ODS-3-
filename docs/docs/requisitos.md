# Requisitos do Projeto

## Requisitos Funcionais

### RF01 — Cadastro de Usuário
O sistema deve permitir que usuários realizem cadastro para acesso à plataforma.

### RF02 — Autenticação
O sistema deve permitir login com credenciais válidas.

### RF03 — Registro de Consumo de Água
O sistema deve permitir o registro diário do consumo de água.

### RF04 — Registro de Atividade Física
O sistema deve permitir o registro de atividades físicas realizadas pelo usuário.

### RF05 — Registro de Sono
O sistema deve permitir o registro de horas de sono.

### RF06 — Definição de Metas
O sistema deve permitir que o usuário configure metas relacionadas aos hábitos monitorados.

### RF07 — Visualização de Dashboard
O sistema deve apresentar indicadores e progresso dos hábitos registrados.

### RF08 — Lembretes e Alertas
O sistema deve emitir lembretes para metas não cumpridas.

---

## Requisitos Não Funcionais

### RNF01 — Responsividade
O sistema deve ser compatível com dispositivos desktop e mobile.

### RNF02 — Desempenho
O tempo de resposta das principais operações deve ser inferior a 3 segundos.

### RNF03 — Segurança
As senhas dos usuários devem ser armazenadas de forma criptografada.

### RNF04 — Disponibilidade
O sistema deve possuir disponibilidade mínima de 95%.

### RNF05 — Usabilidade
A interface deve ser simples, intuitiva e de fácil utilização.

### RNF06 — Escalabilidade
A arquitetura deve permitir evolução e inclusão de novas funcionalidades.
